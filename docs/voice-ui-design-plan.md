## ğŸ”„ **8. User Flows**

### **Flow 1: New Voice Recording**
1. User navigates to `/voice/record`
2. Selects transcription mode (Hybrid recommended)
3. Configures advanced options (Medical AI, Speaker Diarization)
4. Starts recording with real-time feedback
5. Stops recording and processes with ElevenLabs
6. Reviews transcript with medical terms highlighted
7. Saves conversation to history

### **Flow 2: Symptom Entry with Voice**
1. User starts new symptom entry
2. Clicks "Record Voice Description"
3. Records symptom description
4. System processes and populates text field
5. Medical terms are detected and highlighted
6. User can edit transcript if needed
7. Saves symptom with voice recording attached

### **Flow 3: Conversation Review**
1. User navigates to conversation history
2. Searches or filters conversations
3. Selects conversation to review
4. Views transcript with timestamps and speaker labels
5. Plays audio segments or full recording
6. Exports or shares conversation
7. Analyzes medical terms and trends

### **Flow 4: Medical Search**
1. User searches for medical terms
2. System finds relevant conversations
3. Displays search results with context
4. User can play audio segments
5. View related medical terms
6. Export search results
